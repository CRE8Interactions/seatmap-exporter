Overview

What the exporter needs

Deliver a standard .svg using Blocktickets class naming. Named hierarchy is required for reliable parsing — do not rely on unlabeled Illustrator/Figma exports.

The parser looks for sec-* groups. If none are found, a fragile color/geometry fallback may run. Always use the named structure below for production maps.

Required hierarchy

svg
└── g.sec-{SECTION} YZ|NZ        ← one group per sellable area
    ├── g.sec-{SECTION}-row-{ROW}  ← direct child of section
    │   ├── path …-row-…-path       ← optional row hit area
    │   └── rect/path …-seat-{N}    ← seats as direct children of row
    └── g.identifier                ← section cover polygon
        └── path
Critical: Rows and seats must be direct children of their parent. Extra wrapper <g> layers between section → row or row → seat will break parsing.

YZ vs NZ

YZ Yes Zoom

Reserved seating. Rows and seats are parsed. Use for bowl and table sections.

NZ No Zoom

GA / club / suites. Cover path only — seats are ignored even if present.

Naming

Classes, IDs & tokens

Use these exact patterns on class (preferred) or id. Section / row / seat numbers must stay consistent through the tree.

Element Pattern Example
Reserved section sec-{N} YZ class="sec-101 YZ"
GA / non-zoomable sec-{N} NZ class="sec-ga NZ" or id="sec-ga_NZ"
Row group sec-{N}-row-{R} sec-101-row-9
Seat sec-{N}-row-{R}-seat-{S} sec-101-row-9-seat-12
Accessible seat add _DA sec-101-row-9-seat-3_DA
Row hit path sec-{N}-row-{R}-path optional, often opacity="0.01"
Cover group identifier class="identifier" / id="identifier-101"

Token rules

  • No hyphens inside {N}. Use 101, ga, club, suites — not north-101.
  • Match the parent. A seat named sec-102-… inside sec-101 is dropped.
  • Underscores in IDs are OK. id="sec-ga_NZ" normalizes to class="sec-ga NZ".
  • Optional display name: data-seatmap-section-name="Club Level" on the section group.
Special NZ names: ga → “GA Floor”, club → “Club”, suites → “Suites”. Everything else becomes “Section {N}” unless overridden.

Geometry & styling

Shapes, colors, transforms

Seat geometry drives click targets. Brand blue is stripped from the background PNG so interactive seats stay selectable in admin.

Item Requirement
Root SVG Prefer viewBox. Also include width / height when possible.
Seats Prefer <rect> with x, y, width, height. <path> seats also work.
Seat fill Exactly #3358D4
Section cover Path under g.identifier: fill #3358D4, stroke #fff, stroke-width ~3.5
Stage / arena art Non-blue decoration (kept in the background PNG)
Transforms Only rotate(180 x y) / rotate(-180 x y) supported; pivot must equal seat x/y
Not parsed as seats <circle>, <ellipse>, <text>, <use>, nested seat wrappers

Typical section types

Bowl / reserved (YZ)

  • Rows with seat rects
  • Identifier cover path
  • Optional row hit path

Tables (YZ)

  • Row(s) of seat rects
  • Optional table rect (non-seat token) as cover

GA / club / suites (NZ)

  • Identifier cover only
  • No seats required

Accessible seats

  • Include _DA or class token DA
  • Example: …-seat-3_DA

Example

Minimal valid SVG

Copy this shape as a starting point. Real maps will have more sections, rows, and seats — keep the same nesting and naming.

<!-- Stage / arena art (non-blue) -->
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 800 600">
  <rect x="300" y="250" width="200" height="80" fill="#D9D9D9"/>

  <!-- Reserved section -->
  <g id="sec-101" class="sec-101 YZ">
    <g class="sec-101-row-1" id="sec-101-row-1">
      <rect class="sec-101-row-1-seat-1"
            x="100" y="100" width="8.2" height="8.2"
            fill="#3358D4"/>
      <rect class="sec-101-row-1-seat-2"
            x="112" y="100" width="8.2" height="8.2"
            fill="#3358D4"/>
      <rect class="sec-101-row-1-seat-3_DA"
            x="124" y="100" width="8.2" height="8.2"
            fill="#3358D4"/>
    </g>
    <g class="identifier" id="identifier-101">
      <path d="M90 90 H200 V200 H90 Z"
            fill="#3358D4" stroke="#fff" stroke-width="3.5"/>
    </g>
  </g>

  <!-- GA floor (no seats) -->
  <g id="sec-ga_NZ">
    <g class="identifier" id="identifier-ga_NZ">
      <path d="M320 340 H480 V460 H320 Z"
            fill="#3358D4" stroke="#fff" stroke-width="3.5"/>
    </g>
  </g>
</svg>

Delivery

Checklist & common mistakes

Use this before handing off the file. Validate in seatmap-export preview when possible.

Designer checklist

  • One sec-{N} YZ|NZ group per sellable area
  • Rows are direct children of sections: sec-{N}-row-{R}
  • Seats are direct children of rows: sec-{N}-row-{R}-seat-{S} on blue #3358D4 rects
  • Accessible seats use _DA
  • Cover path lives under g.identifier (blue fill + white stroke)
  • GA / club / suites marked NZ with cover only
  • Stage / arena art is non-blue decoration
  • No hyphens inside the section slug; no extra nesting; avoid non-180 transforms
  • Deliver as .svg (or .svgqc from svg-map-checker)

Common mistakes that break parsing

  • 1. Wrapping rows/seats in extra <g> groups
  • 2. Hyphenated section slugs — sec-north-101 fails
  • 3. Mismatched tokens — seat sec-102-… inside sec-101
  • 4. Cover path not under an identifier group
  • 5. Seats drawn as circles/text without sec-…-seat-… names
  • 6. Wrong blue (anything other than #3358D4)
  • 7. GA marked YZ with no rows/cover — section may disappear
  • 8. Arbitrary transforms / nested group transforms on seats
Validate: In seatmap-export run npm run preview, upload the SVG, and confirm section / row / seat clicks before admin import.
